复制含有随机指针节点的链表

题目：
一种特殊的链表节点类描述如下：

即本来 1 -> 2 -> 3 -> 4 -> null，但是多了个随机指针，1的random指针可能指向3，2的random指针可能指向4等等。
需要：拷贝一个这样的链表的结构，并且链表上的node的random指针也要拷贝下来，并返回新的头部。

思路

  • 思路一（使用map）：
  1. 使用哈希map，将节点克隆下来：（1，1‘）、（2，2’）、（3，3‘），并在克隆的同时，让1’的next指针指向2‘，2’的next指针指向3‘。
  2. 现在我们看各节点random指针指向谁？遍历哈希表：假设拿到（2，2’），看2的random指针指向谁，就让2’的random指针指向谁的克隆节点。
  3. 这样遍历下来，就可以设置好所有的random节点，最后返回一个1‘。
  • 思路二（调整链表替代map，不使用额外空间）：
  1. 克隆节点，并放在老节点后面，如做成这样的结构：1 -> 1’ ->2 -> 2’ -> 3 -> 3’ -> null。此时1，2，3的random指向没有变化。
  2. 此时1’，2‘，3’的random指向为null。
  3. 遍历大链表 1 -> 1’ ->2 -> 2’ -> 3 -> 3’ -> null。拿到（1，1‘），1的random我们是知道的，假设为3。此时3的next指向3‘，我们就很容易的将1’的random指针也指向了3‘。
  4. 然后将1’，2‘，3’分离出来。利用奇数头和偶数头，遍历时奇数是老链表，偶数是新链表。
