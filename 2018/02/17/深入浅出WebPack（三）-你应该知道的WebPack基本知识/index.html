深入浅出Webpack（三）-你应该知道的Webpack基本知识

发表于 2018-02-17 | 分类于 深入浅出Webpack

Webpack 更迭

  • v1: 编译打包、HRM（热更新）、代码分割、文件处理。
  • v2: Tree Shaking（打包引用的代码时，会去除未使用的代码）、ES module、动态 Import（是一个动态的Import函数）、新的更人性化的文档。
  • v3: Scope Hoisting（作用域提升，提升了打包后的代码性能。老版本的webpack 将代码打包在闭包当中，这些封装的闭包会损耗浏览器性能。因此。webpack学习他的对手rollup，将所有的代码的模块的作用域，提高到单一的作用域中，这样就提升了打包后代码的运行速度。）、Magic Comments （配合动态import使用，可以使得webpack懒加载一段代码。可以指定打包后的chunk的名字）。

版本迁移

  • V1 -> V2 配置文件不兼容。
  • V2 -> V3 承诺尽可能兼容API。
  • 因此，webpack的配置正在趋于稳定。

Webpack核心概念

Entry

  1. 是代码入口：可以理解为某一段代码的入口。通过代码的入口，直接或间接找到所需要的所有依赖。
  2. 是打包入口：告诉webpack，以哪个文件为起点找到依赖，再找到依赖的依赖。
  3. 可以是一个或者多个entry。例如：多页面程序，一个页面是一个入口。单页面程序，将业务代码和框架代码分为两个entry。

OutPut

  1. 打包生成的文件（bundle）。
  2. 可以是一个或多个。
  3. 如何定义一个entry和output对应起来呢？

Loaders（处理JS之外的资源）

  1. 处理文件
  2. 将非JS的资源文件转换为模块

Plugins（通过插件系统赋能）

  1. plugin参与打包的整个过程。
  2. 有打包优化和压缩、配置编译时变量等功能。

名词

  • chunk：块。在webpack中，代表了代码块。
  • Bundle: 一捆。打包后的代码束。
  • Module：模块，例如：loaders处理了一个图片，也可以看做一个模块。处理了css，也可以看做一个模块。
