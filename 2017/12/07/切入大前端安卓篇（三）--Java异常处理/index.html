异常

什么是异常？

  • 错误分为编译期间和运行期间的错误
  • 在程序运行过程中，意外发生的情况，背离我们程序本身意图的表现，都可以理解为异常。

异常的分类

Java通过Throwable以及它的子类对各种异常进行描述的。Throwable是根类，

  • Error是程序中无法处理的错误，表示运行应用程序中较严重的问题。例如：虚拟机错误（virtualMachineError）、内存溢出（outOfMemoryError）、线程锁死（ThreadDeath）。error是不可查的，它们在应用程序的控制和处理能力之外，而且绝大多数是程序运行时不允许出现的状况。即，发生了error，也不能通过异常处理去处理。
  • Exception，是程序本身可以处理的异常。
  1. Unchecked Exception，非检查异常，是编译器不要求强制处理的异常（runtimeException）。
  2. Checked Exception，检查异常。编译器要求必须处理的异常。

异常的处理

在Java应用程序中，异常处理机制为：抛出异常，捕获异常。

  • 方法中出现异常时，方法创建异常对象，交付给运行时系统去处理。
    -运行时系统捕获异常，进入捕获异常环节。运行时系统去寻找异常的处理器，找到后执行相关的处理逻辑。如果没有找到，运行时系统停止，即Java程序就停止了。

捕获异常

  • 对于可查异常（checkEexception）必须捕捉、或者声明抛出。
  • 允许忽略不可查的RuntimeException和Error，以及它们的子类。

异常处理

通过5个关键字实现：try、catch、finally、throw、throws。

  • finally 无论是否发生异常代码总能执行

throw & throws

throws

  • 可以通过throws声明将要抛出何种类型的异常，通过throw将产生的异常抛出。
  • 如果一个方法可能会出现异常，但没有能力处理这种异常，可以在方法生命处用throws子句声明抛出异常。

throw

  • throw 用来抛出一个异常，例如：throw new IOException()

  • throw 抛出的只能够是可抛出类Throwable或者其子类的实例对象。例如：throw new String，这样写是错误的。

  • 自己抛出异常，自己处理

  • 自己抛出异常，调用方处理

throw 作用

程序在执行中，异常处理不仅可以用来规避风险，也可以用来完成一些需求的特定业务处理。

  • try catch 包含throw语句，自己处理

  • throws抛出给，调用方处理

异常链

  • 有时，我们会捕获一个异常后再抛出另一个异常

  • 例如，方法test one() 捕获一个异常信息，再抛出新的异常对象给另一个方法test two()，再抛出给test Three()

  • 这样只会显示最后的异常信息，会丢失前面的异常信息。

  • 这样，通过构造方法带参，以及调用initCause（），可以将异常信息串起来，形成异常链，显示所有封装进异常对象的异常信息。

异常处理的经验

  • 处理运行时异常时，采用逻辑去合理规避同时辅助try-catch处理。
  • 在多重catch块后面，可以加一个catch(Exception)来处理可能会被遗漏的异常。
  • 对于不确定的代码，也可以加上try-catch,处理潜在的异常。
  • 尽量去处理异常，切忌只是简单的调用printStackTrace()去打印输出。
  • 具体如何处理异常，要根据不同的业务需求和异常类型去决定
  • 尽量添加finally语句去释放占用的资源，例如关闭数据库。

throws注意事项

  • 当子类重写父类抛出异常的方法时，声明的异常必须是父类方法所声明异常的同类或子类
